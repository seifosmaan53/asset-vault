# Documentation

This folder contains all project documentation organized by category.

## Structure

- **`bugs/`** - Bug reports, issue tracking, and fix summaries
- **`reports/`** - Test reports, verification reports, and project status documents
- **`setup/`** - Setup guides, configuration instructions, and deployment documentation
- **`planning/`** - Roadmaps, enhancement plans, and project planning documents
- **Root level** - General documentation files (security, copyright, etc.)

## Quick Reference

### Essential Files (in project root)
- `README.md` - Main project documentation
- `CHANGELOG.md` - Version history
- `QUICK_START.md` - Quick setup guide
- `FIRST_TIME_STARTUP.md` - First-time setup instructions
- `COPYRIGHT.txt` - Copyright information

### Setup & Configuration
- `setup/HOW_TO_START.md` - How to start the application
- `setup/DOCKER_README.md` - Docker setup guide
- `setup/SMTP_SETUP.md` - Email configuration
- `setup/PORT_MANAGEMENT.md` - Port configuration
- `setup/ENVIRONMENT_COMPATIBILITY.md` - Environment compatibility info

### Planning & Roadmaps
- `planning/ROADMAP.md` - Feature roadmap
- `planning/ENHANCEMENT_PLAN.md` - Enhancement plans
- `planning/VALUATION_ROADMAP.md` - Valuation and business planning

### Reports & Status
- `reports/PRODUCTION_READY.md` - Production readiness checklist
- `reports/PROJECT_COMPLETION_STATUS.md` - Project completion status
- `reports/TEST_RESULTS.md` - Test results
- `reports/STABILITY_VERIFICATION.md` - Stability verification

### Bug Tracking
- `bugs/` - Contains all bug reports and fix summaries (historical reference)
